About

“많은 데이터를 어떻게 하면 잘 정리할 수 있을까?”

“정리된 것을 주기적으로 업데이트하고 쉽게 찾아보고 이용하기 위해 가장 효율적인 방법은 무엇일까?”

아마 이런 고민을 한 경험이 한번쯤은 있을 듯 합니다. 저 역시 업무가 직무교육이다 보니 이런 류의 고민이 많았는데 여러 시행착오를 거쳐 최종적으로는 GitHub에 정착하고 GitHub Page에서 블로그를 다시 운영하는 걸로 결정했습니다. 시간이 은근히 많이 들어가겠지만 다시 블로그를 운영해 볼려구요!

GitHub Page에서 Jekyll로 기본적 형태의 블로그를 개설하는 건 여러 블로그에서 아주 자세히 설명해 놓았으니 별로 어렵지 않게 할 수 있습니다. 하지만 맘에 드는 테마를 수정해서 사용하려면 약간의 Front-End 지식이 필요한 듯 보이네요.

제 블로그에서 사용하는 테마는 Jekyll Themes 에서 검색해 찾은 Jasper2라는 Theme입니다. Ghost라는 유명한 publishing platform의 Jekyll 버전인데 맘에 들어서 이것 저것 수정해서 사용하고 있습니다. MIT 라이선스여서 감사하는 마음으로 사용중입니다.

Jasper2 theme를 이용하여 GitHub Page에 Jekyll기반의 블로그를 개설하는 방법은 여기를 누르면 살펴보실 수 있습니다. 초기 세팅부터 Travis-CI를 이용한 Build와 배포 자동화까지 설명할 예정입니다.

제 블로그는 Chrome 전용입니다. IE에서는 화면이 완전히 깨집니다. 볼 수가 없어요.

이 블로그는 오프라인에서 진행하고 있는 직무교육을 보조하는 용도로 사용할 계획입니다. 여러가지 단편적인 지식들을 정리해 나갈 예정이고 가끔 강좌도 올릴 계획입니다.

한글을 다시 깨우치는 중입니다. 말로 하던걸 글로 쓰려니 맞춤법이 어렵습니다. 오탈자나 내용상 문제가 있는부분에 대해서 지적, 태클 환영합니다.
